# 📊 GAME_DAY_TOURS – Proyecto de Análisis de Datos

## 📌 Descripción General
GAME_DAY_TOURS es un proyecto de análisis de datos enfocado en mejorar la toma de decisiones en un operador turístico mediante la integración de datos estructurados y no estructurados.  

El proyecto implementa una arquitectura híbrida combinando SQL Server y MongoDB, junto con procesamiento de datos en Python y R, y visualización mediante dashboards en Power BI.

El objetivo es optimizar la eficiencia operativa, analizar el comportamiento de los clientes y generar insights estratégicos basados en datos.

---

## 🎯 Objetivos
- Analizar patrones de comportamiento de clientes y demanda turística  
- Monitorear indicadores clave del negocio (KPIs) como ingresos, reservaciones y satisfacción del cliente  
- Apoyar la toma de decisiones mediante dashboards interactivos  
- Integrar múltiples fuentes de datos en un flujo analítico unificado  

---

## Arquitectura de Datos
El proyecto utiliza una arquitectura híbrida:

- **SQL Server** → Datos estructurados (transaccionales)  
  - Clientes  
  - Reservaciones  
  - Pagos  
  - Logística  

- **MongoDB** → Datos semiestructurados  
  - Reseñas de clientes  
  - Contenido dinámico  

---

## Tecnologías Utilizadas
- **SQL Server** – Diseño y consultas de bases de datos relacionales  
- **MongoDB** – Base de datos NoSQL  
- **Python** – Procesamiento y análisis de datos  
- **R** – Análisis estadístico y manipulación de datos  
- **Power BI** – Visualización y dashboards  

---

## Funcionalidades Principales
  ### Análisis de Datos
  - Limpieza y transformación de datos  
  - Análisis estadístico del comportamiento de clientes  
  - Procesamiento de datos estructurados y no estructurados  
  
  ### Business Intelligence
  - Dashboards interactivos en Power BI  
  - Monitoreo de KPIs (ingresos, reservaciones, clientes)

  ### Gestión de Datos
  - Diseño de base de datos relacional (SQL Server)  
  - Implementación de colecciones NoSQL (MongoDB)
